# Agent Memory 系统架构（2026）

来源: SitePoint "The New Reality of Agent Memory Complete Guide 2026", 多篇 arXiv 综述

## 记忆分类

### 1. Working Memory（工作记忆）
Agent 当前交互的上下文窗口内容。容量受模型上下文长度限制。

特征: 临时、高优先级、当前任务相关。

### 2. Short-term Memory（短期记忆）
当前会话中超出上下文窗口的历史信息，通常压缩/摘要存储。

特征: 会话级别生命周期、结构化存储、按需检索。

### 3. Long-term Memory（长期记忆）
跨会话持久化信息，使 Agent 具备"经验积累"能力。

### 4. Episodic Memory（情景记忆）
具体的过去交互事件：用户问过什么、Agent 做了什么、结果如何。

用于: 个性化、避免重复错误、参考历史决策。

### 5. Semantic Memory（语义记忆）
抽象化的事实和知识：用户偏好、领域知识、概念关系。

用于: 快速事实检索、一致的知识应用。

### 6. Procedural Memory（程序记忆）
"如何做"的知识：工作流模式、成功的工具调用序列、最佳实践。

## 记忆架构模式

### Flat Memory（扁平记忆）
所有记忆存为键值对，用嵌入检索。简单但缺乏结构。

### Hierarchical Memory（层级记忆）
记忆按主题/时间/重要性组织成树形结构。检索更精准但维护复杂。

### Graph Memory（图记忆）
记忆存为知识图谱（实体为节点，关系为边），支持多跳推理。

### Hybrid Memory（混合记忆）
组合多种存储策略：短期用向量检索、长期用图数据库、情景用时间序列。

## 关键设计决策

| 维度 | 选项 | 权衡 |
|------|------|------|
| 存储后端 | 向量DB / 图DB / 关系DB / 文件 | 向量适合语义检索，图适合关系推理 |
| 检索策略 | 嵌入相似度 / 关键词 / 混合 | 嵌入召回高但可能不精确 |
| 遗忘策略 | FIFO / LRU / 重要性评分 / 摘要 | 需平衡保留关键信息与避免上下文膨胀 |
| 更新策略 | 实时 / 批量 / 触发式 | 实时保证新鲜度但增加开销 |

## 实现选型

- **Mem0**: 专注 Agent 记忆，自动提取/去重/优先级排序
- **LangChain Memory**: 内置 conversation buffer/summary/entity 等多种类型
- **向量数据库**: Pinecone, Chroma, Qdrant 用于语义检索
- **图数据库**: Neo4j 用于关系和推理
