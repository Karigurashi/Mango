# Agent Planning（规划）系统

来源: 掘金 "Planning Agent 架构深度解析", CSDN 主流 Agent 模式对比

## 三大规划范式

### 1. ReAct（Reasoning + Acting）
**边想边做**，交替进行推理和行动。

流程: Thought（思考）→ Action（行动）→ Observation（观察）→ 循环。

优势: 灵活性高，适应性好，简单任务成本低。

劣势: 缺乏全局规划，复杂多步任务容易漂移，无法回溯。

适用: 实时交互、信息检索、简单工具调用。

### 2. Plan-and-Execute（先规划后执行）
**先想后做**，规划与执行分离。

流程: 接收任务 → 生成完整执行计划 → 逐步执行 → 完成。

优势: 全局透视，执行有序，审计友好。

劣势: Planning Tax（前期推理开销），计划可能过时（环境变化时僵化）。

改进: Plan-and-Act 变体在执行中允许动态重规划。

适用: 多系统集成、数据迁移、深度研究综合。

### 3. Reflexion（反思迭代）
**会复盘**，执行后评估并改进。

流程: 执行 → 自我评估 → 识别问题 → 调整策略 → 重新执行。

核心组件: Actor（执行者）、Evaluator（评估者）、Self-Reflection（自我反思）。

优势: 持续改进，从错误中学习，适合需要高质量输出的任务。

劣势: 多次模型调用增加延迟和成本，需要明确的评估标准。

## 其他规划策略

### Tree of Thoughts（思维树）
同时探索多条推理路径，用 BFS/DFS 搜索最佳解。

优势: 探索多种可能性，适合需要创造性或非确定性推理的任务。

劣势: Token 消耗极高（探索多条路径）。

### Task-Decoupled Planning（任务解耦规划）
将复杂任务分解为独立子任务，分配给不同 Agent 或 Pipeline。

优势: 专业化处理，可并行执行。

### Dynamic Planning（动态规划）
执行过程中根据中间结果动态调整后续计划。

优势: 适应性强，能应对不确定性。

## 选型指南

| 任务类型 | 推荐范式 |
|---------|---------|
| 简单问答/信息检索 | ReAct |
| 多步骤有依赖的工作流 | Plan-and-Execute |
| 高质量输出要求 | Reflexion |
| 需要探索多种方案 | Tree of Thoughts |
| 跨领域复杂任务 | Task-Decoupled Planning |
